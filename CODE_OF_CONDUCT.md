# Community code of conduct

India Map Studio is committed to a welcoming, respectful, and constructive
community.

## Expected behavior

- Be considerate and assume good intent while discussing ideas.
- Give specific, actionable feedback about the work rather than the person.
- Respect different levels of experience, languages, identities, and
  geographic perspectives.
- Credit data providers and contributors accurately.
- Accept correction when a boundary name, identifier, source, or historical
  description is wrong.
- Keep political and territorial discussions focused on verifiable project
  data and application behavior.

## Unacceptable behavior

- Harassment, discrimination, threats, or personal attacks.
- Publishing another person's private information.
- Deliberately misleading attribution or knowingly submitting restricted data.
- Repeated disruption, spam, or inflammatory political advocacy unrelated to a
  concrete project change.
- Sexualized language or imagery in project spaces.

## Scope

This code applies in repository issues, pull requests, discussions, reviews,
and other spaces where someone represents the project.

## Reporting and enforcement

For an ordinary disagreement, ask maintainers to moderate the relevant issue
or discussion. For a sensitive report, use the private contact method listed
on the repository owner's GitHub profile. Do not expose private details in a
public issue.

Maintainers may edit or remove content, lock conversations, reject
contributions, issue a warning, or temporarily or permanently restrict
participation. Decisions should be proportionate, documented privately where
appropriate, and applied consistently.

Retaliation against someone who reports a concern in good faith is not
acceptable.
