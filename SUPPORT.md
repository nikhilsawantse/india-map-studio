# Support

## Questions and usage help

Use the repository Discussions area for setup questions, map-building ideas,
and general help once Discussions is enabled.

Before posting:

- read README.md and ATTRIBUTION.md;
- try the latest version from the default branch;
- search existing discussions and issues; and
- remove private or sensitive information from sample data.

## Bugs

Use the bug-report issue form for reproducible application defects. Include
the browser, operating system, page URL, reproduction steps, and a screenshot
when useful.

## Boundary corrections

Use the boundary-data issue form. A correction needs an authoritative source,
the affected administrative level, the relevant codes, and the boundary
vintage. The project cannot treat visual similarity alone as evidence.

## Security

Do not report vulnerabilities in public issues. Follow SECURITY.md.

Support is community-provided and has no guaranteed response time.
